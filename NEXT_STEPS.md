## 🌐 STEP 6: Deploy (Optional but Recommended)

### Deploy Frontend (Netlify - Free)
1. Go to https://netlify.com
2. "Add new site" → "Import from Git"
3. Choose your GitHub repo
4. Settings:
   - Base directory: `client`
   - Build command: `npm run build`
   - Publish directory: `client/build`
5. Click Deploy!

### Deploy Backend (Render - Free)
1. Go to https://render.com
2. "New +" → "Web Service"
3. Connect GitHub repo
4. Settings:
   - Root directory: `server`
   - Build: `npm install && npm run build`
   - Start: `npm start`
5. Add environment variables from `server/.env.example`
6. Deploy!

## 🎤 Interview Preparation

**Memorize these answers:**

**Q: "Tell me about SafeCommute"**
A: "SafeCommute is a women's safety platform I built after seeing how uncomfortable women feel traveling alone. It provides real-time location sharing, emergency SOS, and community safety reports. I used React and TypeScript for the frontend, Node.js for the backend, and integrated geolocation APIs. The biggest challenge was ensuring reliability — if someone hits SOS, every millisecond matters."

**Q: "What makes it different?"**
A: "Most safety apps are reactive panic buttons. SafeCommute is preventive — it tracks journeys proactively, shares location automatically with trusted contacts, and includes features like fake calls for uncomfortable situations. It's designed with Indian women's real experiences in mind."

**Q: "What would you improve?"**
A: "I'd add Google Maps integration for route visualization, Firebase for real-time WebSocket updates, and AI-based risk scoring that analyzes crime data and lighting conditions to warn users before they enter risky areas. I'd also integrate with Uber/Ola for ride verification."
